Homework 3: Exponential & Gamma via Poisson Processes and Monte Carlo

Overview & Motivation

Suppose web requests (or API calls) arrive randomly to a server. When arrivals are independent and have a constant average rate over time, a classic model is the Poisson process with rate \(\lambda\) (arrivals per unit time). Two fundamental random variables arise:

  • Inter-arrival time (\(X\)): time from one request to the next. If arrivals follow a Poisson process, then \(X\) follows an Exponential distribution, \(X \sim \mathrm{Exp}(\lambda)\).

  • Waiting time to the \(k\)-th arrival (\(T_k\)): sum of \(k\) inter-arrival times, which yields \(T_k \sim \mathrm{Gamma}(k, \, \lambda)\) (also called Erlang when \(k\) is an integer).

These distributions show up in network traffic (packet/request timing), call centers, radiation detection, reliability (time-to-failure) and queueing models. The Exponential’s memoryless property and the Gamma’s “sum of exponentials” structure are the key features you’ll explore—both analytically and via Monte Carlo simulation.

Goal of this HW. Build intuition for Exponential & Gamma from first principles, verify properties via Monte Carlo, and connect to Poisson counts.

1) From Poisson counts to the Exponential waiting time

In a Poisson process with rate \(\lambda\), “no arrivals in \((0,x]\)” has probability \(e^{-\lambda x}\). The waiting time \(X\) to the next arrival then satisfies \(P(X>x)=e^{-\lambda x}\).

Imagine time flowing continuously, and “events” (like web requests, calls, or radioactive decays) arriving at random times according to a Poisson process with rate \(\lambda\):

  • \(\lambda\) = average number of arrivals per unit time (e.g., 2 arrivals per minute).

The key assumptions of a Poisson process:

  • Events occur independently, and

  • The number of arrivals in any time interval of length \(t\) follows \(N(t)\sim \mathrm{Poisson}(\lambda t)\), meaning that \(\Pr(N(t)=k)=\frac{e^{-\lambda t}(\lambda t)^k}{k!},\ k=0,1,2,\ldots\).

Define the waiting time random variable \(X\) as the time to first arrival. Then, the event \(\{X>x\}\) is exactly the event that no arrivals have occurred in \((0,x]\), i.e. \[ \{X>x\}=\{N(x)=0\}. \] Hence, using the Poisson probability for zero counts, \[ \Pr(X>x)=\Pr\{N(x)=0\}=e^{-\lambda x},\qquad x\ge 0. \]

From this definition, it should be clear that the cdf of \(X\) is \(F_X(x) = 1-e^{-\lambda x}\) and consequently, the probability density function is \(f_X(x) = \lambda e^{-\lambda x}\).

That is, \(X \sim Exp(\lambda)\). Using integration by parts, we can easily show that \(E[X]=1/\lambda\) and \(\mathrm{Var}(X)=1/\lambda^2\).

Memoryless Property of the Exponential Distribution Let \(X \sim Exp(\lambda)\) be a random variable representing the waiting time until the next event in a Poisson process with rate \(\lambda>0\). Then, for any \(s,t>0\), \[P(X>s+t \mid X>s)=P(X>t)\] This means that the probability of waiting more than \(t\) additional time units does not depend on how long you have already waited (\(s\)). In other words, the exponential distribution has no memory of the past.The exponential distribution is the only continuous distribution that satisfies this memoryless property.

2) Monte Carlo: verifying Exponential properties

Suppose web requests arrive to a server according to a Poisson process with rate \(\lambda=0.5\) requests per minute (so the mean inter-arrival time is 2 minutes). Let \(X\) denote an inter-arrival time.

Exercise 2 Generate \(n=100\) samples for web-requests inter-arrival times (Hint: use rexp). Compute the empirical mean and variance and compare them to the theoretical values. Repeat the process for \(n=100,000\). Briefly explain what you observe.

Solution For an exponential distribution the theoretical mean and variance is described below.

Theoretical mean E[\(X\)] = 1/\(\lambda\)
Theoretical variance V[\(X\)] = 1/(\(\lambda\))^2

Observations: When the sample size increases to 100K we see the empirical mean and variance get closer to their theoretical values.

# Define the rate parameter (lambda)
lambda <- 0.5
# Define the theoretical mean and variance
theoretical_mean <- 1 / lambda
theoretical_variance <- 1 / (lambda^2)

# Function to perform the simulation and comparison
simulate_and_compare <- function(n_samples, lambda_rate) {
  # Generate n samples for inter-arrival times
  samples <- rexp(n_samples, rate = lambda_rate)

  # Compute the empirical mean and variance
  empirical_mean <- mean(samples)
  empirical_variance <- var(samples)

  # Print results
  cat(sprintf("Results for n = %d samples:\n", n_samples))
  cat(sprintf("  Empirical Mean:    %.4f\n", empirical_mean))
  cat(sprintf("  Theoretical Mean:  %.4f\n", theoretical_mean))
  cat(sprintf("  Difference (Mean): %.4f\n", abs(empirical_mean - theoretical_mean)))
  cat(sprintf("  Empirical Variance:    %.4f\n", empirical_variance))
  cat(sprintf("  Theoretical Variance:  %.4f\n", theoretical_variance))
  cat(sprintf("  Difference (Variance): %.4f\n\n", abs(empirical_variance - theoretical_variance)))
}

# Run for n = 100

3) Gamma as a sum of exponentials

In a Poisson process with rate \(\lambda\), the waiting time between consecutive events is modeled by an exponential random variable \(X_i \sim \mathrm{Exp}(\lambda)\). If we let \[ T_k = X_1 + X_2 + \cdots + X_k, \] then \(T_k\) represents the total waiting time until the \(k\)-th event occurs.

It can be shown that \[ T_k \sim \mathrm{Gamma}(k, \lambda), \] with mean \(\mathbb{E}[T_k] = k/\lambda\) and variance \(\mathrm{Var}(T_k) = k/\lambda^2\). This relationship connects the exponential and gamma distributions: the exponential models the waiting time to the event, while the gamma models the waiting time to the event.
